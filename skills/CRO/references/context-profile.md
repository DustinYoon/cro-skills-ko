# 회사 맥락 프로파일 — 인테이크 · 지속 · 분기 (Company Context Profile)

> **어떤 조언이 맞는지는 이 회사의 맥락이 결정한다.** 그래서 모든 CRO 스킬은 **조언 전에 맥락을 확정**하고,
> 한 번 확정하면 **저장해 재사용**한다(매 턴 다시 묻지 않는다). 이 문서가 그 절차·저장·분기 규약이다.
> 오케스트레이터(`CRO`)가 첫 접촉에서 확정하지만, **직접 호출된 스킬도 워크플로 1단계에서 이 절차를 따른다.**

## 1. 프로파일 스키마 (채울 필드)

| 필드 | 값(예) | 이게 뒤집는 것 |
|:--|:--|:--|
| 회사/제품 | 한 줄 | — |
| **GTM 모션** | PLG·셀프서브 / 세일즈주도 / 하이브리드 | MEDDPICC 필요성, 채널 믹스, 자격심사 깊이 |
| **ACV·딜 규모대** | 트랜잭셔널(<$5K) / 미드마켓($5K~50K) / 엔터프라이즈($50K+) | 아웃바운드 수익성, 자격 깊이, 전문화 |
| **세일즈 사이클** | 짧음(<30일) / 중간 / 김(분기+) | 속도 vs 규율, 포캐스트 방식 |
| **구매 주체** | 단일 구매자 / 위원회(멀티스레드) | 챔피언·EB·MEDDPICC 전체 |
| **회사 단계** | pre-PMF·창업자세일즈 / 초기확장 / 스케일 / 성숙 | 전문화 시점, 시스템 투자, 니치 |
| 타겟 세그먼트 | SMB / 미드마켓 / 엔터프라이즈 + 산업·페르소나 | ICP, 메시지 |
| 시장 성숙도 | 신규 카테고리(전도형) / 성숙(경쟁 다수) | 아웃바운드 차별점, Challenger 필요성 |
| 대체 여부 | 그린필드 / 기존 교체 | 아웃바운드 적합성, "왜 지금" |
| 현재 채널 믹스 | 씨앗/그물/창 대략 % | 리드젠 우선순위 |
| 팀 | 규모, 역할 전문화 수준 | 채용·전문화 |
| 제약/정책 | 할인 정책, 컴플라이언스 등 | 협상, 캠페인 |
| (선택) 핵심 지표 | 승률·커버리지·이탈·페이백 | 파이프라인·포캐스트 |

> **분기를 가장 크게 가르는 5개:** GTM 모션 · ACV대 · 사이클 · 구매주체 · 회사 단계. 시간이 없으면 이 5개만이라도.

## 2. 인테이크 절차 (비침습적 — 심문 금지)

0. **상위 맥락 먼저 흡수.** 인테이크 전에 호스트가 이미 준 맥락을 읽어 프로파일을 최대한 채운다 — `CLAUDE.md`(전역 `~/.claude/CLAUDE.md` + 프로젝트)·`AGENTS.md`·`.cursorrules`·`README`·프로젝트 문서, 그리고 **대화 히스토리**. **이미 아는 항목은 다시 묻지 않는다.** (`portability.md` §3)
1. **먼저 추론.** MCP(기업 속성·ACV·사이클·승률), 코퍼스/첨부에서 채울 수 있는 필드를 **자동으로** 채운다.
2. **갭만 질문.** 남은 핵심 필드만 **한 번에 3~5개 이하**로 묶어 물어본다. 우선순위는 위 5개.
3. **모르면 가정 명시.** 답을 못 받으면 지어내지 말고 **가정을 밝히고 진행**한다 — 예: "가정: 미드마켓·세일즈주도로 진행합니다(다르면 알려주세요)." 멈추지 않는다.
4. **지속(persist).** 확정한 프로파일을 카드로 저장해 다음 스킬·턴에서 **재질문 없이** 재사용한다(§3).
5. **드리프트 재확인.** 대화 중 프로파일과 모순되는 신호가 나오면 **그 필드만** 다시 확인한다.

## 3. 지속 규약 (한 번 파악 → 계속 재사용)

- **읽기 순서(진입 시):** 상위 맥락(`CLAUDE.md`/`AGENTS.md` 등, `portability.md` §3) → `$CRO_CONTEXT_FILE` → `./.cro/context.md` → (없으면) §2 인테이크.
- **내부 메모리:** `./.cro/memory/decisions.md`, `experiments.md`, `accounts.md`가 있으면 질문과 관련된 항목만 읽는다. 자세한 운영 규약은 `revenue-operating-system.md`.
- **경로 해석:** references 절대경로가 안 잡히는 환경(프로젝트 설치·앱 샌드박스)에선 `portability.md` §2 순서로 해석.
- **지속 폴백:** FS·셸이 없으면(예: Claude 앱) 파일 대신 **대화 안에 카드를 명시**하고 그 스레드 동안 재사용, 다음 세션용으로 `CLAUDE.md`에 적어두도록 제안(`portability.md` §4).
- **쓰기(확정 후):** 같은 위치에 카드로 저장/갱신. Bash로 처리:
  ```bash
  CTX="${CRO_CONTEXT_FILE:-./.cro/context.md}"
  [ -f "$CTX" ] && cat "$CTX"    # 있으면 로드해서 재사용, 없으면 인테이크 후 아래로 기록
  mkdir -p "$(dirname "$CTX")"
  cat > "$CTX" <<'EOF'
  # CRO 맥락 프로파일
  회사/제품: <...>
  GTM 모션: <PLG|세일즈주도|하이브리드>
  ACV대: <트랜잭셔널|미드마켓|엔터프라이즈>
  사이클: <짧음|중간|김>
  구매 주체: <단일|위원회>
  단계: <pre-PMF|초기확장|스케일|성숙>
  타겟/시장/대체: <...>
  채널 믹스: 씨앗/그물/창 <...>
  제약: <...>
  (지표): <승률·커버리지·이탈·페이백, 아는 만큼>
  출처: <MCP/문서/사용자 제공/추론/가정>
  갱신: <날짜/근거>
  EOF
  ```
- 카드는 **읽기 우선 재사용, 새 사실이 나오면 갱신**. 사용자가 정한 규칙(예: "우리는 할인 안 함")도 여기 기록해 다음에 존중.
- 중요한 결정사항은 `.cro/context.md`에 길게 누적하지 말고 `.cro/memory/decisions.md`에 한 줄씩 추가한다.

## 4. 분기 기본값 (프로파일 → 각 기능의 기본 스탠스)

`conflict-resolution.md`의 적용 조건을 프로파일로 **자동 적용**한 것. 프로파일이 확정되면 아래 스탠스로 시작한다(예외는 드러내고 조정).

| 맥락 | 기본 스탠스 |
|:--|:--|
| **PLG·트랜잭셔널·짧은 사이클·단일구매자** | 자격심사 경량(활성화·확장 신호로 대체, MEDDPICC 과잉) · 속도/자동화 우선 · 아웃바운드는 상향 세그먼트만 · 전원 PG · 포캐스트는 사용량/전환 기반 · 리드젠은 그물+씨앗 |
| **세일즈주도·엔터프라이즈·긴 사이클·위원회** | MEDDPICC 풀 가동 · 3 Whys·챔피언·EB 필수 · 규율 우선("느리게=빨라진다") · 역할 전문화 · 갭×4 커버리지 · POV 4조건 · 리드젠은 창+씨앗 |

| **pre-PMF·창업자세일즈** | 니치 확정·20인 인터뷰 먼저 · 시스템화 전 실험 · 역할 전문화 대신 **시간 블록** · 딜 규모보다 학습 |
| **신규 카테고리(전도형)** | Challenger·교육형 메시지 · 아웃바운드 차별점 조건 강조 · "왜 지금"으로 현상유지 깨기 |
| **성숙·경쟁 다수** | 차별점·의사결정 기준 통제 · FOMU(실패 두려움) 관리(Dixon) · 할인 대신 지표 연결 |

## 5. 스킬 적용 규칙

- **오케스트레이터:** 라우팅 **전에** 프로파일을 로드/확정하고, 하위 스킬 호출 시 그 프로파일을 함께 넘긴다(또는 카드가 저장돼 있으면 하위 스킬이 자동 로드).
- **직접 호출된 스킬:** 워크플로 1단계에서 카드를 로드. 없으면 **그 스킬에 필요한 축만** 최소 인테이크(전체를 다 묻지 않는다).
- **답변에 반영:** 데이터 소스 표기 다음에 한 줄로 맥락을 밝힌다 — 예: `맥락: 세일즈주도·엔터프라이즈·위원회·스케일 → MEDDPICC 풀·규율 우선`.
