# 호환성 & 호스트 통합 (Portability & Host Integration)

> 이 스위트는 **Anthropic Agent Skills(`SKILL.md`)** 포맷이다. 호스트 제품마다 로딩·파일접근·지속 방식이 달라서,
> 스킬은 (1) 참조 경로를 견고하게 해석하고, (2) **호스트가 이미 준 상위 맥락을 먼저 흡수**하고, (3) 지속이 불가능한 환경에선 대화 내 상태로 폴백한다.

## 1. 제품별 지원 현황 (정직하게)

| 호스트 | 스킬 호출 | 참조/FS | 맥락 지속 | 상태 |
|:--|:--|:--|:--|:--|
| **Claude Code (CLI)** | `~/.claude/skills/` 또는 프로젝트 `.claude/skills/` 자동 로드 | 로컬 FS + 셸 O | `.cro/context.md` 파일 지속 O | **네이티브(주 타깃)** |
| **`~/.claude/skills`를 읽는 기타 CLI/툴** | 동일 | 동일 | 동일 | 대체로 O |
| **Claude 앱(웹/데스크톱)** | Skills 기능으로 **패키지 업로드** 필요(로컬 `~/.claude/skills` 안 읽음) | 샌드박스 — `~/` 경로·로컬 FS 가정이 깨짐. references는 **패키지에 동봉**돼야 함 | 대화 간 파일 지속 X → **대화 내 상태로** | 패키징하면 O, 지속은 대화 한정 |
| **Codex (OpenAI) 등 타 에이전트** | `AGENTS.md` 체계 — `/CRO-*` SKILL.md 자동호출 미지원 | 로컬 FS + 셸 O | 파일 지속은 가능하나 스킬 자동호출 X | **비네이티브** — 참조 자료로 쓰거나 `AGENTS.md` 심으로 변환 |

> **핵심:** Claude Code(및 `~/.claude/skills`를 읽는 CLI)에서 **완전 네이티브**. Claude 앱은 **패키징+업로드**가 필요하고 지속은 대화 한정. Codex는 스킬 체계가 달라 네이티브 호출이 안 되니 참조 자료/AGENTS.md로 활용.

## 2. 참조 경로 해석 규칙 (견고하게)

스킬 본문은 편의상 `~/.claude/skills/CRO/references/<file>`로 표기하지만, **읽을 때는 아래 순서로 존재하는 것을 쓴다**:
1. `~/.claude/skills/CRO/references/` (홈 설치 — 기본)
2. `<프로젝트>/.claude/skills/CRO/references/` (프로젝트 설치)
3. 이 스킬 폴더의 형제 `CRO/references/`, 또는 **패키지에 동봉된 `references/`** (스위트를 통째로 옮기거나 앱에 업로드한 경우)

넷 다 없으면(예: 샌드박스에 미동봉) **내장 지식으로 자문 모드**를 진행하고 "레퍼런스 미탑재라 자문 모드로 진행합니다"라고 밝힌다. 절대경로가 안 잡힌다고 멈추지 않는다.

## 3. 상위 맥락 먼저 흡수 (이미 아는 건 다시 묻지 않는다)

맥락 인테이크(`context-profile.md`) **전에** 호스트가 이미 제공한 맥락을 읽어 프로파일을 최대한 채운다:
- **호스트 메모리/규칙 파일:** `CLAUDE.md`(전역 `~/.claude/CLAUDE.md` + 프로젝트), `AGENTS.md`, `.cursorrules`, `README`, 프로젝트 문서 — 회사·제품·시장·GTM 단서.
- **대화 히스토리 & 호스트 시스템 컨텍스트:** 앞선 턴에서 사용자가 말한 ACV·모션·타깃 등.
- **저장된 카드** `.cro/context.md`(있으면 최우선).

읽은 값은 프로파일에 채우고 **출처를 표기**한다 — 예: `맥락: CLAUDE.md에서 회사·모션 확인, 나머지는 질문`. **이미 상위 맥락에 있는 항목은 재질문하지 않는다.** 없는 축만 묻는다.

## 4. 지속(persist) 폴백

- **FS+셸 가능(Claude Code 등):** `.cro/context.md`에 카드 저장 → 턴·스킬·세션 간 재사용.
- **FS 지속 불가(Claude 앱 샌드박스 등):** 카드를 **대화 안에 요약 블록으로 명시**하고 그 스레드 동안 재사용. 새 대화에선 다시 확정하거나, **호스트 메모리(CLAUDE.md 등)에 적어두도록 사용자에게 제안**(그러면 다음엔 §3으로 자동 흡수됨).
- 원칙: 어느 환경이든 **같은 세션에서 한 번 파악한 걸 다시 묻지 않는다.**

## 5. 다른 제품에 태우는 법

- **Claude 앱:** `skills/`를 zip으로 묶어 Skills로 업로드(references 동봉). 절대경로 대신 동봉 references가 §2의 3순위로 해석된다.
- **Codex/타 에이전트:** `references/*.md`를 프로젝트에 두고 읽게 하거나, 오케스트레이터 라우팅+규약을 `AGENTS.md`로 옮긴다.
- **프로젝트 스코프 설치(팀 공유):** `bash install.sh <repo>/.claude/skills` 로 레포에 설치하면 그 프로젝트에서만 로드되고 커밋으로 공유된다.
