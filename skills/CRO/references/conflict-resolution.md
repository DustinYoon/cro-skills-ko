# 상충 해소 & 맥락 적합 규약 (Conflict Resolution & Context-Fit)

> 모든 `CRO-*` 스킬이 **조언을 내기 전에** 따르는 규약.
> 세일즈 지식은 (1) 계속 쌓이고, (2) 서로 상충하며, (3) **맥락 의존적**이다 — 같은 조언이 어떤 상황엔 맞고 어떤 상황엔 틀리다.
> 스킬은 구루의 말을 복창하지 않는다. **먼저 상황을 확정하고 → 맥락에 맞는 조언을 고르고 → 상충은 숨기지 말고 드러낸다.**

## 1. 맥락 프로파일 먼저 (상황이 진실을 뒤집는다)

조언 전에 아래 축을 확정한다(모르면 질문 또는 MCP/코퍼스로 확인). 이 축들이 "어떤 조언이 맞는지"를 결정한다.

| 축 | 값 | 뒤집는 것 |
|:--|:--|:--|
| GTM 모션 | PLG·셀프서브 / 세일즈 주도 / 하이브리드 | MEDDPICC 필요성, 채널 믹스 |
| ACV·딜 규모 | 트랜잭셔널(저) / 미드마켓 / 엔터프라이즈(고) | 아웃바운드 수익성, 자격심사 깊이, 역할 전문화 |
| 세일즈 사이클 | 짧음(원콜~주) / 김(월~분기+) | 속도 vs 규율, 포캐스트 방식 |
| 구매 주체 | 단일 구매자 / 위원회(멀티스레드) | 챔피언·EB·MEDDPICC 전체 |
| 시장 성숙도 | 신규 카테고리(전도형) / 성숙(경쟁 다수) | 메시지, 아웃바운드 차별점 조건 |
| 회사 단계 | pre-PMF·창업자 세일즈 / 확장기 / 성숙 | 전문화 시점, 시스템 투자 |
| 대체 여부 | 그린필드 / 기존 시스템 교체 | 아웃바운드 적합성, "왜 지금" |

> **각 프레임워크 조언에는 적용 조건이 붙는다.** 조건이 상황과 안 맞으면, 그 조언은 이 케이스에서 "틀린" 조언이다.

## 2. 알려진 상충과 맥락별 해소 (대표 사례)

| 쟁점 | 입장 A (적용 조건) | 입장 B (적용 조건) | 해소 |
|:--|:--|:--|:--|
| **MEDDPICC 엄격함** | 필수 (복잡·위원회·고ACV·긴 사이클) | 과잉 (트랜잭셔널·PLG·셀프서브·원콜) | 딜 유형으로 결정. PLG엔 경량 자격(활성화·확장 신호)으로 대체 |
| **아웃바운드** | 강력 (FI2I 4조건 충족) | "AI 시대 콜드는 죽음" (BoomPop) | **둘 다 맞다.** 무차별 자동화 콜드는 죽고, 표적·차별·관련성·진정성 아웃바운드는 유효. 4조건 + 진정성으로 게이팅 |
| **인바운드 vs 아웃바운드** | "인바운드가 선" | "직접 영업만 효과" | 도그마 배격(FI2I 환상의 조합). 믹스는 ACV·사이클·타겟이 결정 |
| **셀프서비스 vs 세일즈 주도** | 셀프서브로 확장 | 세일즈 주도가 정석 | 이분법 깨고 **병렬 퍼널** — 세그먼트·딜 규모로 라우팅 |
| **딜 규모 상향 vs 작은 딜** | 상향해야 큰 비즈니스 | 작은 고객도 씨앗·추천 가치 | 둘 다. 단 리소스 배분은 LTV(2차 효과 포함)로 |
| **역할 전문화** | 확장기 필수 | 초단순·1~2인·원콜엔 불필요 | 회사 단계·프로세스 복잡도로. 소규모는 시간블록 전문화부터 |
| **채용 인재상** | Roberge 5특성 | 회사마다 다름 | 책 스스로 "공식은 회사마다 다르다" 명시. 그대로 이식 금지, **자사 데이터로 재도출** |
| **속도 vs 규율** | "느리게 해야 빨라진다"(디스커버리 투자) | 속도·효율 우선 | 고ACV·복잡딜엔 규율 우선 / 저가·대량엔 속도·자동화 우선 |
| **긴박감(urgency) 걸기** | McMahon "Why now"·컴펠링 이벤트로 압박 | Dixon: 우유부단한 고객엔 FOMO가 87% 역효과 | **무관심 vs 우유부단 진단 먼저.** 무관심(현상유지)엔 긴박감으로 깬다, 합의 후 재불안(FOMU)엔 압박 금지·JOLT로 안심 |
| **LTV 신뢰도** | FI2I "2차 효과로 LTV 과소평가 말라" | Verna "5년 안 됐으면 LTV 모른다, 페이백 봐라" | 상보. 초기엔 페이백 주기(<3개월), 성숙기엔 2차효과 포함 CLTV. 순진한 단일 LTV 추정 맹신 금지 |
| **파이프라인 생성 주체** | FI2I "AE는 발굴 말고 전문화" | Clay "전원이 PG 책임(Clay Day)" | 회사 단계·모션. 초기·PLG·소규모는 전원 PG, 확장기 엔터프라이즈는 전문화 |

> 이 표는 닫힌 목록이 아니다. 새 상충을 만나면 같은 형식(입장+적용조건+해소)으로 다룬다.

## 3. 상충 해소 프로토콜 (순서대로)

1. **맥락 적합 우선.** 상황과 적용 조건이 맞는 쪽을 택한다.
2. **숨기지 말고 드러낸다.** "A는 X(조건일 때), B는 Y(조건일 때), 당신 상황→**Z 권장(근거)**" 형식. 상반된 조언을 뭉개서 섞지 않는다(mush 금지).
3. **출처·시의성.** 항구적 원칙은 책/프레임워크, 현재 전술·시장 변화(AI, 툴링, 채널)는 최신 코퍼스+웹이 우선. 같은 맥락을 최신 소스가 갱신하면 그쪽을 따르되 긴장 관계를 명시.
4. **최종 심판 = 사용자 데이터 + 제1원리.** 구루들이 갈리면 이름값이 아니라 사용자 CRM/딜 실측(MCP)과 첫 원리로 결정.
5. **불확실하면 실험.** 못 정하면 작은 테스트로 검증한다(FI2I "현실에서 빠르게 테스트", SAF 이론→테스트→피드백→반복).

## 4. 지식 누적 규칙 (Knowledge Currency)

- **안정 코어 = 내장 프레임워크**(`frameworks-*.md`, 항구적 원칙). **진화층 = `CRO_CORPUS_DIR`**(신규·상황적, 때로 상충).
- **코퍼스 ↔ 책 충돌 시:** 맥락이 같고 코퍼스가 더 최신·구체적이면 코퍼스 우선(단, 원칙 자체가 흔들리면 명시). 맥락이 다르면 **둘 다 보존**하고 조건으로 구분.
- **재사용할 결정은 기록.** 사용자가 "우리는 X 상황에선 Y로 간다"를 정하면 코퍼스/노트에 남겨 다음에 재적용(경량 learnings). 스킬은 이를 발견하면 우선 존중.
- 스킬의 말투: "책이 이렇게 말함"이 아니라 **"당신 상황 + 최신 증거 기준 이렇게 권장"**.

## 스킬 적용 규칙 (요약)

모든 CRO 스킬은 조언 전에: **(1) 맥락 프로파일 확정 → (2) 적용 조건 대조 → (3) 상충 시 §3 프로토콜(드러내고 맥락으로 결정) → (4) 데이터/실험으로 심판.**
근거 없는 단정·구루 복창·상반 조언 뭉개기 금지. 확신 없으면 "상황에 따라 다르다"를 명시하고 어떤 상황에 무엇이 맞는지 조건으로 갈라 답한다.
