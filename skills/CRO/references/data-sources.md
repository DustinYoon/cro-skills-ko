# 데이터 소스 해석 규약 (Data-Source Resolution Contract)

> 모든 `CRO-*` 스킬이 따르는 **이식성(portability) 규약**입니다.
> 이 스킬 스위트는 전역(`~/.claude/skills/`)에 설치되어 **누구나, 어떤 디렉토리에서도** 동작해야 합니다.
> 따라서 스킬은 특정 폴더(예: 개인 스터디 코퍼스)를 **절대 가정하지 않습니다.**

## 핵심 원칙

1. **프레임워크 지식은 스킬에 내장(bundled)되어 있다.** `references/frameworks-*.md`가 유일한 지식 원천이며,
   이것만으로 자문(advisory)이 가능하다. 외부 파일이 없어도 스킬은 동작한다.
2. **사용자의 "우리 회사 데이터"는 런타임에 주입한다.** 아래 우선순위로 소스를 해석한다.
3. **하드코딩된 경로 금지.** `CRO 스터디/…` 같은 개인 경로를 코드/프롬프트에 박지 않는다.

## 데이터 소스 우선순위 (위에서부터 시도, 가능한 것으로 폴백)

| 순위 | 소스 | 감지 방법 | 쓰임 |
|:-:|:--|:--|:--|
| 1 | **MCP 서버** (HubSpot·Gong·Salesforce 등) | 현재 세션에 연결된 MCP 툴이 있는가 (`mcp-data-access.md` 참조) | 실시간 파이프라인·콜·리드·딜 데이터 |
| 2 | **사용자 지정 코퍼스 (라이브)** | 환경변수 `CRO_CORPUS_DIR`, 또는 사용자가 명시한 경로/글롭 (예: `리뷰-완료/`·`원문 파일/`의 `*_핵심정리.md`) | 사내/스터디 세일즈 노트·플레이북·인터뷰 요약. **최신 에피소드는 내장 `insights-corpus.md`보다 우선(신선도)** |
| 3 | **첨부/붙여넣기** | 사용자가 대화에 붙인 파일·표·CRM 내보내기(CSV) | 단건 딜/리스트 분석 |
| 4 | **내장 프레임워크만** | 위 셋 다 없음 | 순수 자문 모드(사용자 상황을 질문으로 채움) |

## 스킬이 반드시 지키는 규칙

- **소스를 먼저 밝힌다.** 답변 시작에 "데이터 소스: [MCP:HubSpot] / [코퍼스:<경로>] / [첨부] / [자문 모드]"를 1줄로 표기.
- **없으면 만들어내지 않는다.** MCP·코퍼스·첨부가 없으면 수치를 지어내지 말고, 필요한 입력을 **질문으로** 요청하거나 자문 모드로 진행한다.
- **자문 모드에서도 유용해야 한다.** 데이터가 없어도 프레임워크로 "무엇을 확인해야 하는지 / 어떤 질문을 해야 하는지 / 어떤 순서로 움직여야 하는지"를 준다.
- **소스 혼합 가능.** 예: MCP로 딜 데이터 + 첨부로 통화 메모 → 함께 분석.

## `CRO_CORPUS_DIR` 규약 (선택)

사용자가 사내 세일즈 자료(플레이북, 인터뷰, 요약)를 스킬에 물리고 싶을 때:

```bash
export CRO_CORPUS_DIR="/path/to/our-sales-notes"   # 예: 회사 위키 export, 개인 스터디 폴더
```

- 스킬은 이 디렉토리를 **읽기 전용 참고 자료**로만 쓴다(자동 수정 금지).
- 없으면 조용히 건너뛴다(에러 아님).
- 이 저장소의 원저작자처럼 `CRO 스터디/리뷰-완료/` 같은 개인 코퍼스가 있다면 여기에 물리면 된다 — **opt-in**.

## 진입 시 소스 점검 스니펫 (스킬이 참고용으로 실행)

```bash
# 1) MCP 연결 여부는 세션 툴 목록으로 판단 (mcp-data-access.md 참조)
# 2) 사용자 코퍼스
if [ -n "$CRO_CORPUS_DIR" ] && [ -d "$CRO_CORPUS_DIR" ]; then
  echo "CORPUS: $CRO_CORPUS_DIR"
else
  echo "CORPUS: (none — advisory or MCP/attached only)"
fi
```

> 요약: **지식은 내장, 데이터는 주입.** 이 규약 덕분에 스킬은 개인 폴더 없이도 전역에서 동작하고,
> 원하는 사람은 자기 데이터(MCP 또는 코퍼스 경로)를 꽂아 즉시 개인화할 수 있다.
