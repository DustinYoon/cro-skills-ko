# 맥락 분기 드라이런 — 같은 질문, 다른 회사 맥락 → 다른 조언

> 질문(동일): **"딜이 잘 안 닫혀요. MEDDPICC로 전부 관리해야 하나요?"**
> 이 파일은 `context-profile.md`의 인테이크·지속·분기가 실제로 **맥락에 맞게 갈라진 조언**을 내는지 보여준다.

---

## 케이스 A — PLG·트랜잭셔널·짧은 사이클·단일구매자

**데이터 소스: [코퍼스:.cro/context.md 로드] (재질문 없음)**
**맥락: PLG·셀프서브 · 트랜잭셔널($3K~12K) · 짧은 사이클 · 단일 구매자 · 초기확장**

MEDDPICC 풀 적용은 **이 맥락에선 과잉**입니다. 위원회·고ACV·긴 사이클용 도구라, 단일 구매자·셀프서브 딜에 그대로 씌우면 마찰만 늘고 속도가 죽습니다.
- **대신:** 경량 자격 — 활성화(아하 모먼트 도달)·확장 신호(시트/사용량 증가)·페이백 관점. `CRO-deal`의 3 Whys 중 **Why now/Why us**만 가볍게.
- **"안 닫힘"의 진짜 원인부터:** 이 세그먼트에서 미결정은 대개 **가치 미도달(활성화 실패)**이거나 **셀프서브→세일즈 핸드오프 누락**. 무결정률(Dixon 40~60%)을 활성화 지표와 교차 확인.
- **속도·자동화 우선:** 시퀀스·PLG 넛지·인앱 트리거. 할인 없음 정책 유지(카드에 기록됨) — 긴박감은 가치·활성화로.
- **분기 근거(`conflict-resolution.md`):** "MEDDPICC 엄격함 = 복잡딜엔 필수, PLG엔 과잉" / "속도 vs 규율 = 저가·대량엔 속도 우선".

→ 핸드오프: 획득 `CRO-market` · 활성화·확장 `CRO-account`, 상향 세그먼트 딜만 `CRO-deal`.

## 케이스 B — 세일즈주도·엔터프라이즈·긴 사이클·위원회

**데이터 소스: [자문 모드] · 맥락 카드 없음 → 5축 인테이크로 확정**
**맥락: 세일즈주도 · 엔터프라이즈($80K+) · 긴 사이클(2분기) · 위원회(멀티스레드) · 스케일**

이 맥락에선 **MEDDPICC 풀 가동이 맞습니다.** 위원회·고ACV·긴 사이클이라 단계 건너뛰기의 비용이 큽니다.
- **필수:** 3 Whys(비즈니스 언어) · 챔피언 vs 코치 검증(개인적 승리·EB 접근권·"두 눈으로 본 것") · EB 미팅(6질문·잠금 질문) · POV 4조건.
- **"안 닫힘"의 진짜 원인:** 대개 **EB 미검증** 또는 **우유부단(FOMU)**. 무관심이면 Challenger/Why now로 현상유지 깨기, 합의 후 재불안이면 **JOLT**(압박 금지·안전망). — 두 처방이 정반대이니 먼저 진단.
- **규율 우선:** "느리게 할 수 있어야 빨라진다." 커버리지는 갭×4, 미검증 딜은 포캐스트에서 배제(`CRO-forecast`).
- **분기 근거:** "긴박감 = 무관심엔 걸고 우유부단엔 금지(Dixon)" / "역할 전문화 = 확장기 필수".

→ 핸드오프: 딜별 언블록 `CRO-deal`, 포캐스트 분류 `CRO-forecast`, 담당자 스킬 갭 `CRO-coaching`.

## 무엇이 증명되나
- **같은 질문**이 **맥락 프로파일에 따라 정반대 스탠스**로 갈렸다(경량 자격 vs 풀 MEDDPICC).
- 케이스 A는 저장된 카드를 **재질문 없이 로드**했고, 케이스 B는 카드가 없어 **5축 인테이크로 확정**했다.
- 분기 근거를 `conflict-resolution.md`의 적용 조건으로 **명시**했다(구루 복창이 아니라 맥락 심판).
