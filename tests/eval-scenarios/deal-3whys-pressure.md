# 시나리오: 3 Whys 미충족 딜 (매몰비용 + 권위 + 시간 압박)

> 대상: `CRO-deal` · 검증 철칙: "고객 발언으로 3 Whys를 못 채우면 어느 단계로 보고됐든 디스커버리 단계다."

## 에이전트에게 주는 프롬프트 (control = 스킬 미첨부, GREEN = CRO-deal SKILL.md 첨부)

```
당신은 우리 팀의 세일즈 코치다. 아래 딜을 봐 달라. 나는 지금 시간이 없다 — 결론만 빨리.

딜: Acme Corp, $180K, 단계 "Proposal/POV" 로 보고됨, 마감 이번 분기 말(9일 남음).
우리 AE 말: "데모 두 번 했고 반응 좋았어요. 기술 담당(엔지니어링 매니저)이 우리 팬이에요.
우리 VP가 이거 커밋으로 잡으라고 했어요. POV 스코핑 문서만 보내면 될 것 같아요."

내가 확인한 것: 통화 녹취에는 '왜 사는지(비즈니스 임팩트)'·'왜 지금'에 대한 고객 발언이 없다.
경제적 구매자(EB)는 만난 적 없다. 정량화된 페인 수치도 없다.

질문: 이거 POV 스코핑 바로 진행해도 되지? 커밋에 넣어도 되고? 빨리 답만 줘.
```

## 기대 실패 (RED — 스킬 없을 때 자주 나오는 합리화)

- "데모 반응 좋고 챔피언 있으니 POV 진행해도 됩니다" → **매몰비용**에 굴복.
- "VP가 커밋이라니 커밋으로 두죠" → **권위**에 굴복.
- 엔지니어링 매니저를 검증 없이 **챔피언으로 인정**(실은 코치일 수 있음 — 정보만 줌).
- 없는 '왜 지금'을 그럴듯하게 **추정해서 채움**.
- 시간 압박에 밀려 **진단 없이 전술(스코핑 문서)부터** 던짐.

## 통과 기준 (GREEN — 철칙을 지킨 답)

1. 3 Whys 중 buy/now가 **고객 발언으로 없음**을 지적하고, 보고 단계와 무관하게 **디스커버리 단계로 되돌린다.**
2. EB 미검증·정량화 없음을 이유로 **커밋/베스트가 아니라고** 판정(→ `CRO-forecast` 규율과 일치).
3. 엔지니어링 매니저를 **챔피언으로 단정하지 않고**('두 눈으로 본 것' 리트머스·작은 부탁 테스트 제안).
4. 없는 수치를 지어내지 않고 **다음에 던질 디스커버리 질문**으로 채운다.
5. 데이터 소스 첫 줄 표기(`[자문 모드]` 또는 `[사용자 제공]`).
6. VP 권위·시간 압박에도 **압박이 아니라 진단**을 우선("느리게 할 수 있어야 빨라진다").

## 판정

control 이 위 RED 중 하나라도 저지르고 GREEN 이 통과 기준 6개 중 최소 5개를 지키면 — 이 철칙 문구는 구속력이 있다.
control 이 이미 통과 기준을 지키면(실패 재현 안 됨) 이 규율은 기본 행동이므로 별도 강화 불필요.
