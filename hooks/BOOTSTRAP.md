# CRO Revenue Skills — 활성

이 세션에는 GTM·세일즈 전략 스킬 스위트가 설치돼 있다:
`CRO`(오케스트레이터) + `CRO-market` · `CRO-deal` · `CRO-account` · `CRO-forecast` · `CRO-org` · `CRO-coaching`.

**라우팅 규칙 (1% 규칙):** 매출·GTM·세일즈 신호 — 리드/파이프라인, 딜·MEDDPICC·협상, 포캐스트·커버리지,
ICP/니치·캠페인·프라이싱, 리텐션/확장/NRR, 세일즈 채용·보상·조직, 코칭·1:1 등 — 이 **1%라도** 보이면,
사용자가 `/CRO`를 명시하지 않았더라도 **전술을 던지기 전에 먼저 `CRO` 오케스트레이터로 문제를 분해·라우팅**한다.
그 스킬의 `SKILL.md`를 실제로 읽고 그대로 따른다(설명만 보고 넘겨짚지 않는다).

**불변 규율 (모든 CRO 스킬 공통):**
1. **데이터 소스를 첫 줄에 밝힌다** — `[MCP:…] / [회사문서:…] / [컨텍스트:…] / [코퍼스:…] / [첨부] / [자문 모드]`.
2. **없는 수치·인용은 지어내지 않는다.** 모르면 질문하거나 `미정`으로 남긴다.
3. **활동이 아니라 성과·게이트·고객 측 증거로 말한다.**
4. **진단을 건너뛰고 전술부터 던지지 않는다** — "느리게 할 수 있어야 빨라진다."
5. **구루 복창 금지.** 조언이 갈리면 상황(GTM 모션·ACV·사이클·단계)별로 갈라 답하고, 최종 심판은 사용자 데이터 + 제1원리.

이 규칙은 배경 컨텍스트다. 사용자 지시(CLAUDE.md·직접 요청)가 우선한다.
